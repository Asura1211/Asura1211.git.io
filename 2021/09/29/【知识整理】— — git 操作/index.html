【知识整理】— — git 操作

发表于2021-09-29|更新于2021-11-21|知识整理
|字数总计:592|阅读时长:2分钟|阅读量:

因为每次使用 git 操作都需要搜索一番，好多流程记不住顺序，就打算在这里记录一下使用 git 时的一些常用操作以及经常遇到的问题和解决方案吧~

创建仓库&&拉取项目

部分参考自博客

1、单击 New repository

2、输入仓库名（英文），添加 Description（可选），设置仓库类型，选择 Initialize this repository with a README，点击 **Create repository **就可以了。

3、在文件夹 git bash ，通过 git init 生成 .git 文件

3、配置本地仓库的账号邮箱git：

$ git config --global user.name "Your Name"

$ git config --global user.email "email@163.com"

4、为了避免每次远程访问需要输密码，所以使用ssh登陆。ssh应该与本机信息绑定。查看自己电脑C:\Users\Administrator 目录下是否有 .ssh 文件夹。如果没有就需要生成：$ ssh-keygen -t rsa -C "email@163.com"。ssh 在 Github 中备份后才能被验证。在 github 的 MyProfile 中，点击 Add Public Key

5、key 中的内容在 C:\Users\Administrator 里面的 .ssh文件夹，打开 id_rsa.pub，全部复制内容到 key

6、克隆项目：$ git clone https://github.com/***.git

克隆时可能出现：fatal: unable to access 'https://github.com/......': OpenSSL SSL_read: Connection was reset, errno 10054，原因是服务器的SSL证书没有经过第三方机构的签署。

解决办法：

  • 输入git config –global http.sslVerify “false”，解除ssl验证
  • 将地址https://改为git://，避开ssl验证

7、git checkout -b xxx 新建分支

将本地的代码关联到github上（如果是第一次上传代码）： git remote add origin git@github.com:***.git

更新本地代码到仓库

1、将自己写的代码添加到暂存区： git add .

2、可以查看暂存区状态变化信息： git status

3、添加上传的注释（必须要有注释内容哦）： git commit -m “上传的版本信息”

4、上传代码到 github 之前需要先 pull： git pull origin main

5、上传代码到远程仓库： git push -u origin main

在 git push 时可能会出现下面的错误：

fatal: unable to access ‘https://github.com/***.git/': OpenSSL SSL_read: Connection was reset, errno 10054

产生原因：一般是因为服务器的 SSL 证书没有经过第三方机构的签署，所以才报错

解决方法：git config --global http.sslVerify "false"

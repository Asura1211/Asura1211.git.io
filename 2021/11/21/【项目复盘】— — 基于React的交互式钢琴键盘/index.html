“项目复盘”

发表于2021-11-21|更新于2021-11-22|项目复盘
|字数总计:569|阅读时长:1分钟|阅读量:

介绍

本项目基于 React 和 ice.js 实现了一个交互式钢琴键盘及按键可视化展示效果，主要包括以下功能：

  • 弹奏钢琴，记录用户操作历史

  • 播放用户弹奏历史

  • 播放用户输入的音乐

  • 动态显示当前音频柱状图及随机音符

分工

  • 项目搭建，主要代码编写

  • 代码编写，组件拆分

  • 组织分工，难点指导，debug

  • debug，材料编写

  • 项目线上部署，后期维护

难点或亮点阐述

难点：

  1. 钢琴琴键的布局问题：由于黑键和白键交替布局，且黑键叠于白键之上，若布局不当便无法正常显示正常钢琴，在此处黑键使用绝对定位，使其脱离文档流，便可正常显示；

  2. 组件化与全局状态问题：由于钢琴、波形图、音符元素的触发均依赖于按键历史这一变量，但将页面组件化后，各组件间不能直接通信，于是定义状态模型，将全局状态存储在 store 中，实现不同组件间的状态共享；

  3. 实现音符的上升效果：若音符匀速上升，动画效果较为单一，音符路径较为单一；若音符位置改变的时间跨度太大，则会出现类似于”卡顿“的现象。针对以上两种情况，采用了递增音符上升速度、缩小时间跨度以增大更新率的解决方案，得到了良好的展示效果；

  4. 页面自动部署问题：项目采用 gh-pages 部署，但由于网络问题，在使用命令行部署时经常会出现请求超时等问题，后采用 github action 自动部署，解决了这一难题。

亮点：

  1. 解决方案采用 icejs，开箱即用，减少项目前期的配置，支持 Vite 构建模式，更好支持热更新，减少构建时间；

  2. 项目页面特效展示与输出依赖于用户操作，交互性强，用户能得到良好的使用体验；

  3. 项目自定义程度高，门槛低，不同用户能体验到不同的项目使用方式，满足不同的项目需求。

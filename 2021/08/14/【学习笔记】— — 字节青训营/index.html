动态修改元素与样式

一、Elements

  • DevTools 里面 Styles 中点击 .cls 开启动态修改元素的 class，输入字符串可以动态的给元素添加类名。
  • Computed 下点击样式里的箭头可以跳转到 styles 面板中的 css 规则。
  • 修改动态样式（:hover）时，可以如第一点，点击 :hov ；或者 DOM 元素右击 -> Force state -> :hover。

二、Console

  • 灵活使用 log、warn、error、debug、info 查看日志。（在控制栏左侧选择等级，对日志分类查看）
  • console.table 具象化展示 JSON 和数组数据。
  • console.dir 展示对象中的属性与值（处理 DOM 节点）。
  • 占位符：给日志添加样式，突出重要的信息。如：console.log('%c%s', 'font-size:24px; color:red', 'Welcome!');
    • %s：字符串占位符
    • %o：对象占位符
    • %c：样式占位符
    • %d：数字占位符

四、Performance

TypeScript

‘ TS 的类型检查’与‘生成 JS ’是两个独立的过程；类型检查出错不影响生成 JavaScript 代码。

类型别名：用 type 声明类型，与 let 类似，采用块级作用域。

Node.js

简介

  1. 异步I/O：当Node,js 执行I/O 操作时，会在响应返回并恢复操作，而不是阻塞线程并浪费CPU循环等待。

  2. 单线程：Node.js保持了JavaScript在浏览器中单线程的特点。

    优点:

    1. 不用处处在意状态同步问题，不会发生死锁
    2. 没有线程上下文切换带来的性能开销

    缺点:

    1. 无法利用多核CPU
    2. 错误会引起整个应用退出，健壮性不足
    3. 大量计算占用导致CPU，无法继续执行

Representational State Transfer 翻译过来是”表现层状态转化”，它是一种互联网软件的架构原则。因此复合
REST风格的Web API设计，就称它为RESTful API

RESTful特征:

  • 每一个URI代表一种资源(Resources), 比如: http://kaikeba.com/courses ;
  • 客户端和服务器之间，传递这种资源的某种表现层，比如：http://kaikeba.com/courses/web;
  • 客户端通过HTTP动词，对服务器端资源进行操作，实现”表现层状态转化”，比如：POST http://kaikeba .com/courses

二、开发

即一些 bundle（打包工具）

webpack

原理：递归地构建一个依赖关系图(dependency graph)，其中包含应用程序需要的每个模块，然后将所有这些模块打包成一个或多个 bundle。

rollup

天然支持 esm，（导入的模块没用，则不会在 bundle.js 中出现）

1、原理：通过 esm 生成静态 ast，分析代码得到数据结构，再生成对应的代码。解析代码后会得到很多的信息，基于这些信息做处理，比如合并代码，删减代码（tree-shaking）。

2、plugins 的用法 plugins: [json()]，比如

  • 解析 json ： import json from "@rollup/plugin-json";
  • 压缩生成的 bundle 文件：import {terser} from 'rollup-plugin-terser'

且在不同的输出文件中支持不同的规范。

3、rollup 天然不解析依赖，遇到只有 commonjs 规范的库也解析不了

rollup 和 webpack 区别

  • 写库时 rollup 更好（因为纯代码，不需要什么资源）
  • 开发应用时 webpack，因为有很多优化如热更新、devserver…

9.24 Vue3

设计理念

  • 易上手
  • 数据驱动 （减少 dom 操作）：声明式渲染，vdom，patch
  • 渐进式（progressive）：min、vuex、router…
  • 设计动机
    • 用函数的方式创建：类型支持，消灭 this。（函数：vue3、react；class：vue2、Angular）
    • 实例方法和属性（tree-shaking）：消灭 dead code。比如 Vue.component('comp',{})。
    • 复用性、可读性、可维护性：数据和控制数据的业务写在一起，可以封成一个 hook。
    • API 简化：一致性；简化，删除一些相同功能的 API（比如 v-model 与 .sync），还有 render。
    • 扩展性：自定义渲染器， Vue.createRenderer()
    • 性能优化：响应式，基于 Proxy。
      • vue2 Object.defineProperty(obj,{}) 基于递归，效率问题
      • 数组无法用 defineProperty，只能单独实现
      • api 影响，动态属性的新增删除 Vue.delete/set
      • class map set 等的支持问题
      • 但是 Proxy 的兼容性很差（IE11也无法支持）

setup 优先级高于 data。当两者中有属性重复时，显示 setup 的。
